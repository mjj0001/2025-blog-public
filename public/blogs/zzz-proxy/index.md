### 项目代码回顾梳理

依旧为 Python + Frontend 框架

所以整体目录为：

```md
- frontend
- python
```

做为前端，使用前端脚本为主，所以根目录 `package` 文件放置命令， 并且根目录放置 `scripts` 文件夹放置 安装依赖/开发/构建 命令
```md
- frontend
- python
- scripts
**package.json**

> 额外的静态内容
- .venv               python依赖文件夹
- node_modules  前端依赖文件夹
- exe                 打包成 exe 的资源文件夹
- .vscode           编辑器配置文件夹，重要的：如开始启动快捷键

加上一些零散配置文件
```

### 服务规划

基于上次的情况，这次我将功能进行解耦，作为单例线程自运行，并且性能调优不放在用户侧，这样能安排性能释放。设立专门的 `services` 目录

比如：

* **窗口捕捉**服务，自运行，单例自动存入最新帧、时间戳
* 游戏**页面匹配**服务，自运行，单例自动存入最新匹配帧数据
* **日志**服务，独立运行，接入前端
* **后端**服务，单独文件夹，提供联系前端

再将各自独立的 工具/临时服务，放入 lib 文件夹。

再分别创建 `runtime` 运行时文件夹，包含 **任务**单例服务、自动**辅助**页面单例服务、运行**状态**单例服务

最后是 `task` 目录，专门放置各种任务，运行时服务感知不到具体任务，由任务函数自动调用不同任务程序。
